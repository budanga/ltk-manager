# Phase 1 Data Model — Per-Mod WAD Footprint Visibility

## Entities

### `ModWadReport` (transport + in-memory)

Returned by `OverlayBuilder` (upstream) and by the `get_mod_wad_report` / `analyze_mod_wads` Tauri commands.

| Field                    | Type (Rust)     | Type (TS)                    | Notes                                                                                  |
| ------------------------ | --------------- | ---------------------------- | -------------------------------------------------------------------------------------- |
| `mod_id`                 | `String`        | `string`                     | Mod identifier matching `LibraryModEntry.id`                                           |
| `affected_wads`          | `Vec<String>`   | `string[]`                   | Game-relative WAD paths, sorted lexicographically, deduplicated                        |
| `wad_count`              | `u32`           | `number`                     | Convenience: `affected_wads.len()` (sent over the wire to avoid recomputation)         |
| `override_count`         | `u32`           | `number`                     | Total override entries the mod contributes (across all layers)                         |
| `content_fingerprint`    | `Option<u64>`   | `bigint \| null` (via ts-rs) | From `ModContentProvider::content_fingerprint`                                         |
| `game_index_fingerprint` | `u64`           | `bigint` (via ts-rs)         | Hash of the `GameIndex` snapshot used during computation                               |
| `computed_at`            | `DateTime<Utc>` | `string` (ISO 8601)          | When the report was produced                                                           |
| `is_stale`               | `bool`          | `boolean`                    | Computed at read time by comparing fingerprints to the current values; never persisted |

**Validation rules**

- `wad_count == affected_wads.len()` (asserted before persist).
- `affected_wads` MUST contain no duplicates and MUST be sorted (canonical form for diffing).
- `mod_id` MUST exist in the current library when persisted; reconcile drops orphans.

### `CachedWadReport` (persisted form)

Same fields as `ModWadReport` minus `is_stale` (which is recomputed on read). Persisted as JSON inside `wad-reports.json`.

### `WadReportStore` (Rust, in-memory)

```rust
pub struct WadReportStore {
    path: PathBuf,
    version: u32,
    reports: BTreeMap<String, CachedWadReport>,
    current_game_index_fp: Option<u64>,
}
```

**Methods** (per constitution: methods, not free functions):

- `WadReportStore::load(path: PathBuf) -> AppResult<Self>` — tolerates missing/corrupt file by starting empty (FR-012).
- `WadReportStore::save(&self) -> AppResult<()>` — temp-file-then-rename.
- `WadReportStore::get(&self, mod_id: &str) -> Option<ModWadReport>` — fills `is_stale` against `current_game_index_fp` and the report's own `content_fingerprint`.
- `WadReportStore::upsert(&mut self, report: ModWadReport) -> AppResult<()>` — replaces existing entry, persists.
- `WadReportStore::upsert_many(&mut self, reports: Vec<ModWadReport>) -> AppResult<()>` — single save at the end (used by the patch piggyback path).
- `WadReportStore::invalidate_by_content(&mut self, mod_ids: &[String])` — used by the reconcile pass when archive content fingerprints drift. Batched for efficiency.
- Note: `current_game_index_fp` is updated implicitly when new reports are upserted; no explicit setter is needed.
- `WadReportStore::prune_orphans(&mut self, valid_ids: &HashSet<String>)` — drops reports for uninstalled mods.

### `WadReportState` (Tauri-managed)

```rust
pub struct WadReportState(pub Mutex<WadReportStore>);
```

Locked via the existing `MutexResultExt` pattern.

### File: `wad-reports.json`

```json
{
  "version": 1,
  "current_game_index_fp": "12345678901234567890",
  "reports": {
    "skin.aatrox.bloodbeast": {
      "mod_id": "skin.aatrox.bloodbeast",
      "affected_wads": ["DATA/FINAL/Champions/Aatrox.wad.client", "..."],
      "wad_count": 3,
      "override_count": 47,
      "content_fingerprint": "9876543210987654321",
      "game_index_fingerprint": "12345678901234567890",
      "computed_at": "2026-04-08T12:34:56Z"
    }
  }
}
```

`u64` fields are serialized as numeric JSON values. The TypeScript bindings (generated by ts-rs) represent them as `bigint`. These fingerprints are only used for equality comparisons, never arithmetic, so precision is not a concern in the Rust/JSON layer. The frontend receives them opaquely and does not inspect their values.

## State Transitions

```
                ┌──────────────┐
                │   ABSENT     │  (no entry in cache)
                └──────┬───────┘
       analyze /        │
       patch success    │
                        ▼
                ┌──────────────┐
                │   FRESH      │  (fingerprints match current)
                └──────┬───────┘
        archive edit / │  game update
        external mod   │
                        ▼
                ┌──────────────┐
                │   STALE      │  (rendered with stale indicator)
                └──────┬───────┘
       analyze /        │
       patch success    │
                        ▼
                ┌──────────────┐
                │   FRESH      │
                └──────────────┘
```

`ABSENT` is also reached from any state via mod uninstall (`prune_orphans`).

## Relationships

- `WadReportStore.reports[mod_id]` ↔ `LibraryIndex.entries[mod_id]` — 0..1 cached report per library mod. Reports without a matching library entry are pruned on reconcile.
- A `ModWadReport` is independent of profiles and layer state — it represents the mod's full potential WAD footprint, not the currently-enabled subset (per spec edge case).

## Validation derived from spec FRs

| FR                               | Enforced by                                                                       |
| -------------------------------- | --------------------------------------------------------------------------------- |
| FR-001 (load-order independence) | Per-mod metadata is collected before the cross-mod merge step in `ltk_overlay`.   |
| FR-002 (persistence)             | `WadReportStore::save` after every `upsert` / `upsert_many`.                      |
| FR-008 (content staleness)       | `is_stale` derived from `content_fingerprint` mismatch on read.                   |
| FR-009 (game staleness)          | `is_stale` derived from `current_game_index_fp` mismatch on read.                 |
| FR-010 (auto-refresh on patch)   | Patch piggyback path always overwrites the entry.                                 |
| FR-012 (corrupt cache tolerance) | `WadReportStore::load` falls back to empty store on parse error and logs at WARN. |
